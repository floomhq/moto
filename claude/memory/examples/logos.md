# Brand Logo Sourcing Template

How to find and use real, high-quality brand logos instead of AI-generated or text-based placeholders.

## The Rule

**NEVER use text-in-circles for brand logos.**

Text in a circle is immediately recognizable as placeholder/AI slop. Real brands have distinctive mark designs. Use real SVGs.

## Priority Order (Use in This Order)

### 1. Simple Icons (BEST)
- **URL**: https://simpleicons.org/
- **Format**: SVG icons, 24x24 recommended
- **Coverage**: 1000+ brands, very high quality
- **License**: CC0 (public domain), safe for commercial use
- **How to use**:
  ```bash
  # Download SVG directly
  curl https://cdn.simpleicons.org/github/000000 > github.svg
  # Replace hex with brand color or 000000 for black
  ```

### 2. SVGL (Good Alternative)
- **URL**: https://svgl.app/
- **Format**: SVG logos, 24x24 to 128x128
- **Coverage**: 500+ brands
- **License**: Mixed (check per logo)
- **How to use**: Search, copy SVG code directly into your component

### 3. Gilbarbara's Logos (Comprehensive)
- **URL**: https://github.com/gilbarbara/logos
- **Format**: SVG and PNG, 200px+ high quality
- **Coverage**: 1000+ brands
- **License**: MIT + brand owner policies
- **How to use**:
  ```bash
  # Clone repo (large, ~500MB)
  git clone https://github.com/gilbarbara/logos.git

  # Find logo
  find ./logos -iname "*github*"

  # Copy to your project
  cp logos/brands/svg/github.svg src/assets/logos/
  ```

### 4. Brand Favicons (Last Resort)
- **URL**: `https://www.google.com/s2/favicons?domain=github.com&sz=128`
- **Format**: PNG, usually 16x16 or 32x32
- **Coverage**: Any domain with a favicon
- **Limitations**: Low resolution, might be outdated
- **How to use**:
  ```jsx
  <img src={`https://www.google.com/s2/favicons?domain=${domain}&sz=128`} alt="logo" />
  ```

## Never: AI Generation / Text Circles / Cheap Placeholders

| Anti-Pattern | Problem | Solution |
|--------------|---------|----------|
| Text-in-circle (AI) | Looks cheap, lazy, unprofessional | Use SimpleIcons SVG |
| Gradient circles (AI) | Screams "generated by AI" | Use real logo |
| Font-based logo text | Not scalable, looks wrong | Use brand's actual SVG mark |
| Placeholder emoji | Lazy for production | Spend 30 seconds sourcing real logo |
| Screenshot of logo | Pixelated, copyright risky | Download high-res SVG instead |

## Implementation Examples

### React Component with SimpleIcons

```jsx
// logos.js
export const logos = {
  github: 'https://cdn.simpleicons.org/github/181717',
  slack: 'https://cdn.simpleicons.org/slack/e01e5a',
  figma: 'https://cdn.simpleicons.org/figma/f24e1e',
};

// Usage
<img src={logos.github} alt="GitHub" width={24} height={24} />
```

### React Component with Local SVG

```jsx
import GitHubLogo from './logos/github.svg';

export function BrandIcon({ brand }) {
  const icons = {
    github: GitHubLogo,
    // ...
  };
  return <img src={icons[brand]} alt={brand} />;
}
```

### Next.js Image Component (Optimized)

```jsx
import Image from 'next/image';

export function BrandLogo({ brand, size = 24 }) {
  return (
    <Image
      src={`/logos/${brand}.svg`}
      alt={brand}
      width={size}
      height={size}
      priority={false}
    />
  );
}
```

## Brand Color Reference

When using SimpleIcons, include official brand colors:

```jsx
const brandColors = {
  github: '#181717', // GitHub black
  slack: '#e01e5a', // Slack pink
  figma: '#f24e1e', // Figma orange
  twitter: '#1da1f2', // Twitter blue
};

// Usage
<img
  src={`https://cdn.simpleicons.org/github/${brandColors.github}`}
  alt="GitHub"
/>
```

## Anti-Pattern: When NOT to Use SimpleIcons

SimpleIcons uses brand colors, but sometimes you need monochrome:

```jsx
// For dark mode, monochrome version
<img
  src="https://cdn.simpleicons.org/github/ffffff" // White SVG
  alt="GitHub"
/>
```

## Troubleshooting

### SVG Not Displaying
- Check MIME type: `Content-Type: image/svg+xml`
- Verify viewBox is set correctly
- Check for external resource blocks (CSP)

### Color Not Changing
- SimpleIcons URL format: `https://cdn.simpleicons.org/BRAND/HEX`
- HEX without `#`: `ffffff` not `#ffffff`
- Some brands have licensed marks (be careful with color changes)

### Permission/License Questions
- SimpleIcons: CC0, safe to use anywhere
- Gilbarbara: Check LICENSE for each brand
- Official favicons: Usually fair use for display/linking
- When in doubt, link to the official brand page instead of embedding

## Summary Workflow

1. Search SimpleIcons first (https://simpleicons.org/)
2. If not found, check SVGL (https://svgl.app/)
3. If still not found, clone gilbarbara/logos repo
4. Last resort: favicon or ask for clarification on brand
5. Store logos in `/public/logos/` with source documentation
6. Never generate or use placeholder circles
